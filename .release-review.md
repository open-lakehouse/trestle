# Release review guidance

Repo-specific focus for the `release-notes` skill. Mechanics (crates, tags,
changelog grouping) come from `release-plz.toml` — this file is review intent only.

## Cross-crate couplings to watch

- **Codegen → example sync.** The golden-path example
  (`examples/golden-path-app/`) is regenerated from `olai-codegen` output and is
  verified against the checked-in state in CI. A change to `olai-codegen` usually
  implies a regen plus edits to `examples/golden-path-app/README.md` and the
  `examples/` narratives — check those when codegen bumps.
- **WASM transport seam.** Generated clients pick a transport via
  `transport_type_path`; `olai-http-wasm::WasmClient` is the browser backend
  (`transport: wasm`). Client/transport changes typically touch **both**
  `olai-codegen` (the generated client) and `olai-http-wasm` (the backend) — a
  bump to one without the other is worth a second look.
- **`olai-trestle → olai-codegen`** is the only intra-workspace dependency.
  release-plz syncs the version automatically; just confirm the CLI docs
  (`crates/trestle/README.md`) still match any codegen-surface change.

## Invariants to flag in the heuristic cross-check

- **Credential redaction.** Credential structs in `olai-http` must never
  `#[derive(Debug)]`; they hand-write `impl fmt::Debug` rendering `<redacted>`
  for every secret-bearing field. Flag any diff under
  `crates/olai-http/src/**/credential.rs` (or new credential types) that adds a
  derived `Debug` or a field that could leak a secret.
- **No platform coupling.** This is a generic framework — flag any new hardcoded
  reference to a specific data platform (Unity Catalog, Hive, etc.) in library
  code or docs.

## User-facing surfaces to drift-check

- The `trestle` binary name, `trestle.yaml` config keys, and `TRESTLE_*` env
  vars are the stable user contract (the crate is *published* as `olai-trestle`
  but the binary stays `trestle`). Confirm README/CLI docs match any change here.
- Proto annotation semantics (`google.api.resource`, `field_behavior`,
  `resource_reference`, `debug_redact`, `google.api.http`) are documented in
  `crates/olai-codegen/README.md` and the `trestle-codegen` skill — keep them in
  sync when codegen changes what an annotation emits.

## Highlights voice

Terse and user-facing. Lead with what a capability *unlocks* for someone building
a proto-driven service ("you can now generate a browser client", "pick your
protobuf runtime"), not the internal mechanism. Crates are early `0.x` — always
surface breaking changes and the exact migration, even on a minor bump.
